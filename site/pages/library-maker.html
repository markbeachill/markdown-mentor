Markdown Library Maker

Make one clean Markdown file from many source files

WhatA general-purpose source-packing tool. It makes one structured Markdown library file from a file, folder, or ZIP.
WhyAI tools work better when source material is clearly labelled, separated, and traceable.
NextUse it first when you want to give an AI a clean source library.

Markdown Library Maker is not only for teaching. You can use it for research, project notes, policy documents, writing sources, meeting packs, or any other set of files you want to turn into one AI-readable Markdown file.

Plain name

In the teaching workflow, the same kind of file may be called an Educational Content Training Pack. The general name is simpler: a Markdown library file.

Commands

Make a new Markdown library file

markdown-library make ./sources -o my-library.md

Add more files later

markdown-library add my-library.md ./more-sources

List the sources inside the library

markdown-library list my-library.md

Check the source markers

markdown-library check my-library.md

What the file contains

  • a short header
  • the purpose, if you gave one
  • clear source-start and source-end markers
  • source file names and paths
  • source fingerprints
  • converted Markdown text
  • a separate source manifest next to the library file

When to use Markdown Mentor instead

Use Markdown Mentor when your purpose is teaching or learning. Markdown Mentor starts from a Markdown library file and helps you check, plan, generate, and export teaching materials.

Technical appendix

The command-line entry point is markdown-library. The implementation lives mainly in markdown_mentor/build_pack.py and markdown_mentor/library_cli.py.

  • build_library() makes a new library file.
  • add_to_library() adds sources to an existing library.
  • list_library_sources() lists recorded sources.
  • check_library_format() checks source markers.
