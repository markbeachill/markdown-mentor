Workflow

From a folder of files to finished materials

WhatThe whole workflow once, with the exact commands you type and the prompts you paste.
WhySeeing the full chain once makes each step clear. After this you can do the same with your own files.
NextInstall the tool first (see the Install page), then follow these steps in order.

1. Collect sources YOU

Put the files you want to teach from into one folder. The sample project already has three small files in examples/sample-sources. For your own work, see the source guide in the Library.

2. Build the content pack SOFTWARE

This converts every file into one Markdown file the AI can read, plus a manifest listing what it found.

$ markdown-mentor build-pack examples/sample-sources -g "teach thesis statements"

3. Check the pack SOFTWARE

This writes a readiness note that flags thin coverage, duplicates, or possible private data. It never stops you; it makes risks visible.

$ markdown-mentor check-pack content-pack.md -g "teach thesis statements"

4. Decide what to teach YOU + AI

Write a Teaching Brief, or paste the Suggest teaching briefs prompt into a chatbot with your content pack and edit one of its suggestions. A Teaching Brief says what to teach, who for, the level, and what to include or leave out.

5. Choose how to teach it YOU

Pick a Pedagogy Specification from the Library. This sets the style: tone, level, scaffolding, practice, and feedback. ("Scaffolding" means the support you give learners, which you reduce as they improve.)

6. Choose which materials to make YOU

Pick a Teaching Materials List, such as Getting Started or Revision Pack. This is the set of material types you want for one purpose.

7. Plan the materials AI, CHECKED BY YOU

Paste the Generate the materials inventory prompt into a chatbot, with your content pack, brief, pedagogy, material specifications, and materials list. The AI returns a numbered plan. This is the main checkpoint: fixing the plan now is far easier than fixing many finished documents.

8. Generate each material AI, CHECKED BY YOU

For each inventory item, paste the Generate one teaching material prompt into a chatbot. The AI returns one material in Markdown. Read it, edit anything wrong, and save it as a .md file in a folder such as materials.

9. Export SOFTWARE

Turn your Markdown materials into the formats people use.

$ markdown-mentor export materials -f docx -o exports $ markdown-mentor export materials -f pptx -o exports $ markdown-mentor export materials -f html -o exports

To control the look, add a style profile from the Library:

$ markdown-mentor export materials -f docx -o exports -s library/style-profiles/student-handout.json

PDF

To make a PDF, use -f pdf. This uses LibreOffice (a free office suite) if you have it installed. If you do not, the tool tells you, and you can open the Word or web file and use "Save as PDF" in your own software.

Shortcut

To be walked through every step in the terminal, run markdown-mentor start. It runs the software steps for you and points you to the right prompt for each AI step.
