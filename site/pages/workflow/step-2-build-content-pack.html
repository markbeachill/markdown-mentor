Workflow / Step 2

Step 2

Make the Markdown library file

Who leads this step: Markdown Library Maker

Teaching and learning view

This step turns scattered source files into a stable shared source library. The learner may never see this file directly, but it gives the AI a clearer base for planning and drafting.

WhatMarkdown Library Maker reads the source files, converts supported formats to Markdown, and writes one library file plus a source manifest.
WhyAI tools work better when source material is clearly separated, labelled, and easy to check. A library file reduces the chance that one source blurs into another.
NextRead the manifest to see what was included, then let Markdown Mentor check the library for teaching use.

What to do

  1. Make a Markdown library file from your source folder.
  2. Open the manifest and check that important files were included.
  3. Open the library file and scan the headings and source boundaries.
  4. Fix missing or wrongly converted sources before moving on.

Check before moving on

Ask: does the Markdown library file represent the sources clearly enough for an AI to use without confusion?

Technical appendix

This is the practical detail for running or finding the step in the repo.

Command to copy

markdown-library make examples/sample-sources -o content-pack.md -p "teach thesis statements"

Teaching alias

markdown-mentor build-pack examples/sample-sources -g "teach thesis statements"

Add sources later

markdown-library add content-pack.md ./more-sources

Files and details

  • The general output is a Markdown library file.
  • In the teaching workflow, this file can also be called an Educational Content Training Pack.
  • The manifest is written next to the library file, for example content-pack-manifest.md.
  • Each source is wrapped in clear source-start and source-end markers.
  • The build step records source paths, file sizes, and checksums in the manifest.
