Step 2

Build the content pack

Who leads this step: Markdown Mentor

Teaching and learning view

This step turns scattered teaching files into a stable shared text. The learner does not see this file directly, but it gives the AI a clearer base for planning and drafting.

WhatMarkdown Mentor reads the source folder, converts supported files to Markdown, and writes one content pack plus a source manifest.
WhyAI tools work better when source material is clearly separated, labelled, and easy to quote from. A content pack reduces the chance that one file blurs into another.
NextRead the manifest to see what was included, then check whether the pack is good enough for the teaching goal.

What to do

  1. Run the build command on your source folder.
  2. Open the manifest and check that important files were included.
  3. Open the content pack and scan the headings and source boundaries.
  4. Fix missing or wrongly converted sources before moving on.

Check before moving on

Ask: does the content pack represent the sources clearly enough for an AI to use without confusion?

Technical appendix

This is the practical detail for running or finding the step in the repo.

Commands

$ markdown-mentor build-pack examples/sample-sources -g "teach thesis statements"

Files and details

  • The content pack is usually written to content-pack.md.
  • The manifest is usually written to content-pack-manifest.md.
  • Each source is wrapped in clear source-start and source-end markers.
  • The build step records source paths, file sizes, and checksums in the manifest.
