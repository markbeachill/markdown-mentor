Step 3

Check the pack

Who leads this step: Markdown Mentor

Teaching and learning view

This is a teaching quality gate. It does not judge the subject perfectly, but it helps you notice obvious problems before the AI turns them into worksheets, slides, or quizzes.

WhatMarkdown Mentor writes a readiness note that flags likely issues such as thin coverage, duplicates, missing practice, or possible private data.
WhyA weak content pack can still produce confident AI output. Checking first helps keep the final learning material honest and useful.
NextRead the readiness note. If it raises serious risks, improve the source folder or narrow the teaching brief.

What to do

  1. Run the check command with the same teaching goal.
  2. Read the readiness note before using any AI prompt.
  3. Treat warnings as prompts for teacher judgement, not as automatic pass/fail rules.
  4. Use the improve-content-pack prompt if you want AI help deciding what to add.

Check before moving on

Ask: can the pack support the teaching goal, or should the goal be narrower?

Technical appendix

This is the practical detail for running or finding the step in the repo.

Commands

$ markdown-mentor check-pack content-pack.md -g "teach thesis statements"

Files and details

  • The check is rule-based. It is not an AI judgement.
  • The note is usually written to content-pack-readiness.md.
  • The checker looks for signals such as word count, number of sources, examples, questions, assessment language, duplicates, and possible private data.
  • Useful prompt: Improve the content pack.
