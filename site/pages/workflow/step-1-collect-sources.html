Workflow / Step 1

Step 1

Collect sources

Who leads this step: You

Teaching and learning view

This step decides what the learner can fairly be asked to learn. Good teaching materials need more than background reading: they need explanations, examples, practice, and a clear sense of the learner level.

WhatPut the files you want to teach from into one folder. Use files that are useful for teaching, not just files that are interesting to you.
WhyThe AI can only plan and draft from the source material you give it. If the folder is thin, too advanced, contradictory, or missing examples, the final materials will show those weaknesses.
NextCreate a source folder, then make the Markdown library file from that folder.

What to do

  1. Start with the teaching goal. Write one plain sentence: “teach X to Y so they can Z.”
  2. Add core explanations, worked examples, practice questions, assessment criteria, diagrams, and learner-level notes where you have them.
  3. Remove duplicates, private learner data, and files you do not have permission to use.
  4. Keep background reading only if it helps the actual teaching task.

Check before moving on

Ask: could a teacher or tutor build a useful lesson from these files without guessing too much? If not, add better sources before continuing.

Technical appendix

This is the practical detail for running or finding the step in the repo.

Files and details

  • examples/sample-sources contains a small demo folder.
  • Source folders can contain nested folders and ZIP files.
  • Markdown Library Maker relies on MarkItDown for conversion, so install MarkItDown before using Word, PDF, PowerPoint, or HTML sources.
  • Useful library file: library/source-guides/collecting-sources.md.
