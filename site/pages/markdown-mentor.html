Markdown Mentor

Use a Markdown library file to make teaching materials

WhatThe education-focused workflow. It checks source material, guides the AI steps, and exports finished Markdown materials.
WhyTeaching materials need focus, level, pedagogy, practice, and teacher review. A source library alone is not enough.
NextFirst make a Markdown library file. Then use the workflow to decide what to teach and what materials to make.

Markdown Mentor keeps the human educator or learner in control. The software checks and exports. The AI proposes, plans, and drafts. You decide, edit, approve, and teach.

Commands

Start the guided workflow

markdown-mentor start

Check a library/content pack for teaching use

markdown-mentor check-pack my-library.md -g "teach thesis statements"

Export finished Markdown teaching materials

markdown-mentor export ./materials -f docx

Teaching alias for making a content pack

markdown-mentor build-pack ./sources

Recommended wording

For general users, say “make a Markdown library file”. For the education workflow, you can add: “In Markdown Mentor, this library file is used as the Educational Content Training Pack.”

Technical appendix

The command-line entry point is markdown-mentor. The teaching alias build-pack is kept for compatibility, but the general source-packing command is markdown-library make.

  • run_guide() runs the guided workflow.
  • check_pack() writes the teaching readiness note.
  • export_path() exports finished Markdown files.
  • build_pack() is the education-facing alias for making a content pack.
