Normal route: download one setup file into a new folder, run it, then put your source files in 1-source-files/.

Download one setup file

Download this file into a new folder for your project:

Download setup-markdown-mentor.py

Run it in that folder

python setup-markdown-mentor.py

The setup file downloads the two working scripts and creates the project folders. It only changes the folder where you run it. It does not upload your files.

Files the setup script downloads

  • make-markdown-library.py
  • make-teaching-materials.py

Direct links

https://markbeachill.github.io/markdown-mentor/downloads/setup-markdown-mentor.py
https://markbeachill.github.io/markdown-mentor/downloads/make-markdown-library.py
https://markbeachill.github.io/markdown-mentor/downloads/make-teaching-materials.py

After setup

  1. Put your source files in 1-source-files/.
  2. Run python make-markdown-library.py make.
  3. Follow the prompt files in the project folders.
  4. Export with python make-teaching-materials.py export docx.

Developer route

The repository install route is for developers and testers, not most users.

pip install -e ".[dev]"
