Install

Set up the early prototype

This is not yet a normal app with a double-click installer. These instructions are for testers and technical users. Ordinary teachers and learners should not need to do this in the final version.

WhatYou will install Python, MarkItDown, Markdown Library Maker, and Markdown Mentor.
WhyPython runs the tools. MarkItDown reads Word, PDF, PowerPoint, HTML, and text files.
NextFollow the tester route below. Use the technical notes only if you already know the command line.

Important

You do not need to understand code, but you do need to run a few commands. A command is a line of text you copy into a command window and run by pressing Enter.

Step 1: Install Python

Python is the free program that runs the prototype.

  1. Go to python.org/downloads.
  2. Download the current Python 3 version.
  3. Install it.
  4. On Windows, tick Add Python to PATH before you click Install.

Step 2: Get the project folder

For testers, the simplest route is to download the project as a ZIP file from GitHub.

  1. Open the Markdown Mentor GitHub page.
  2. Click the green Code button.
  3. Click Download ZIP.
  4. Open your Downloads folder.
  5. Extract the ZIP file.
  6. Open the extracted folder until you can see README.md and pyproject.toml.

That folder is the project folder. Some commands must be run from that folder because Python needs to see pyproject.toml.

Step 3: Open a command window in the project folder

Use the method for your computer.

Windows

Open the folder that contains README.md. Click the address bar at the top of File Explorer. Type cmd. Press Enter.

Mac

Open Terminal. Type cd with a space after it. Drag the project folder into the Terminal window. Press Enter.

Linux

Open the project folder in your file manager. Right-click in the folder and choose Open in Terminal, if your file manager has that option.

Step 4: Check Python

Command to copy

python --version

If you see something like Python 3.12.1, Python is ready. If the command is not found, try this instead:

python3 --version

Step 5: Install MarkItDown

Command to copy

python -m pip install "markitdown[all]"

If you used python3 earlier, use this instead:

python3 -m pip install "markitdown[all]"

Step 6: Install the local prototype

Run this from the project folder, the folder that contains README.md and pyproject.toml.

Command to copy

python -m pip install -e .

If you used python3 earlier, use this instead:

python3 -m pip install -e .

Step 7: Check it worked

Check Markdown Library Maker

markdown-library --version

Check Markdown Mentor

markdown-mentor --version

If both commands show a version number, the prototype is installed.

Try the two tools

Make a source library

markdown-library make examples/sample-sources -o content-pack.md

Start the teaching workflow

markdown-mentor start

Optional: PDF export

To export straight to PDF, install LibreOffice, a free office suite. If you do not want to install it, export to Word or HTML and use your usual software to save as PDF.

If something goes wrong

  • The command is not recognised. Python may not be installed correctly. Reinstall Python and tick Add Python to PATH on Windows.
  • python does not work. Try python3 instead.
  • python -m pip does not work. Python may not have installed pip. Try reinstalling Python from python.org.
  • You are in the wrong folder. The command python -m pip install -e . only works in the folder that contains README.md and pyproject.toml.
Technical appendix

The prototype currently requires Python 3.10 or newer. It installs as an editable local package with python -m pip install -e .. MarkItDown is installed from PyPI with the optional [all] extras so it can read more file types.

Useful commands:

python --version python -m pip install "markitdown[all]" python -m pip install -e . markdown-library --version markdown-mentor --version
